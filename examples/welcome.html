A programming language that compiles to native code.

Thin C++/LLVM compiler. Scheduler, HTTP, sync, and I/O in a runtime written in xlang. C/Go-flavored syntax — no VM, no garbage collector.

Features

Static types, modules, and a global runtime linked without imports.

Native codegen

The compiler lowers sources to LLVM IR and links with Clang. Single static binaries.

Runtime in xlang

print, spawn, fetch, file I/O — implemented in the language, embedded at build time.

HTTP & fetch

Router, middleware, JSON/HTML responses. HTTPS client via OpenSSL bridge.

Modules & libs

Directory packages under libs/ — http, json, test, process.

Sync primitives

Lock, RWLock, AtomicInt — embeddable in structs, worker pools via spawn.

Time API

Now(), Format(), ElapsedMs() — globally available, TypeScript Date-like.

Example

HTTP server with the built-in router. Full reference in LANGUAGE.md.

  • Static typing — int32, string, struct, array
  • Variadic print, function overloading
  • Vitest-style test runner (*.test.xlang)
  • VS Code extension with IntelliSense
examples/http_server.xlang 
import * as http from http

fn handle_ping(ctx: Context) {
    ctx.String(200, "pong")
    return 0
}

fn main() {
    local r = http.NewRouter()
    r.Get("/ping", handle_ping)
    r.ListenAndServe("0.0.0.0", 8080, on_listen)
    return 0
}

Runtime

Syscalls are an OS bridge only — pthread, sockets, files. Scheduling and protocol logic live in runtime/*.xlang.

Get started

Clone, build, run.

git clone https://github.com/hasirciogluhq/xlang
cd xlang && mkdir build && cd build
cmake .. && cmake --build . -j$(nproc)
./xlang run ../examples/hello.xlang
