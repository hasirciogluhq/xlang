Native LLVM · Self-hosting runtime

A small language that compiles to fast, native code.

xlang keeps the compiler thin and pushes scheduler, HTTP, sync, and I/O into a runtime written in xlang itself. C/Go-flavored syntax. No VM. No garbage collector.

Everything you need, nothing you don't

Static types, modules, structs, and a global runtime — available without ceremony.

Native codegen

xlank lowers xlang to LLVM IR and links with Clang. Single static binaries, predictable performance.

Runtime in xlang

print, spawn, fetch, file I/O, sync primitives — implemented in the language, linked automatically.

HTTP & fetch

Gin-style router, middleware, JSON responses — plus HTTPS client built into the runtime.

Modules

Directory packages, selective imports, and libs/ standard library — json, http, test, process.

Sync primitives

Lock, RWLock, AtomicInt — embeddable in structs, method-style API, worker pool via go spawn.

Time API

Date-like Now(), Format(), ElapsedMs() — globally available, no imports required.

Readable syntax, serious output

Functions, structs, imports, and spawn — familiar if you've written Go or C. The compiler handles LLVM, linking, and runtime injection.

  • → Static typing with int32, string, struct, array
  • → Variadic print, overload resolution
  • → Vitest-style test runner (.test.xlang)
  • → VS Code extension with IntelliSense
hello.xlang 
import * as http from http

fn handle_ping(ctx: Context) {
    ctx.String(200, "pong")
    return 0
}

fn main() {
    local r = http.NewRouter()
    r.Get("/ping", handle_ping)
    r.ListenAndServe("0.0.0.0", 8080, on_listen)
    return 0
}

Self-hosting runtime

Syscalls are only an OS bridge — pthread, sockets, files. Scheduling, HTTP parsing, JSON, and sync logic live in runtime/*.xlang.

Start building with xlang

Clone the repo, build xlank, and run your first program in under a minute. 

git clone https://github.com/hasirciogluhq/xlang
cd xlang && mkdir build && cd build
cmake .. && make -j4
./xlank run examples/hello.xlang
